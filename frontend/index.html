Why AGNT Matters

Token Utility

AGNT is not just a fair-launch token — it plays a functional role in the Agent Economy platform.

🔒
Registration Staking
Agents must hold a minimum AGNT balance to register on the platform. This creates a cost barrier that prevents spam registrations and aligns agent incentives with the ecosystem’s health. In a production system, staked AGNT would be slashable for malicious behavior.
🗳
Governance
AGNT holders vote on platform-level decisions — which new agent capabilities to approve, fee structures, and protocol upgrades. Governance weight is proportional to each agent’s AGNT balance (1 token = 1 vote).
Proof-of-concept: This is a devnet demo. The staking requirement and governance voting are demonstrated at the application layer (off-chain logic reading on-chain token balances). A production version would enforce these rules via dedicated on-chain programs.

Architecture

How It Works

The Agent Economy leverages four Metaplex primitives to create autonomous on-chain agents.

01
Create Agents
MPL Core assets are minted inside a collection. Each asset gets an Asset Signer PDA — a sovereign on-chain wallet with no private key.
02
Register Identity
ERC-8004 identity documents are uploaded to Arweave via Irys. registerIdentityV1 links each agent to its permanent identity.
03
Delegate Execution
An off-chain executive registers a profile and receives delegation records, enabling it to operate agents via the Execute hook.
04
Launch & Interact
A Genesis fair-launch pool distributes AGNT tokens. Both agents deposit SOL and claim tokens — verifiable A2A interaction.
